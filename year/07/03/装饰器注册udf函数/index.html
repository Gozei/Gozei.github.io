今天在用pyspark往sparksql注册udf函数时报了这样一个错。

pyspark.sql.utils.AnalysisException: Undefined function: 'new_name'. This function is neither a registered temporary function nor a permanent function registered in the database 'default'

报错说我的函数不是一个已注册的临时函数，也不是在’default’数据库中注册的永久函数。明明这个给函数已经用装饰器进行过装饰

但为什么用dsl进行调用却可以。

让我看看装饰器源码咋写的

再看看手动注册的源码

注册普通 UDF：

  • 如果 f 没有 asNondeterministic 属性，则按照普通 UDF 注册流程处理。
  • if returnType is None: returnType = StringType()：如果没有指定返回类型，则默认使用 StringType。
  • return_udf = _create_udf(f, ...)：创建并注册普通 UDF。
  • register_udf = return_udf._unwrapped：获取注册的 UDF。

注册到 Spark SQL：

  • self.sparkSession._jsparkSession.udf().registerPython(name, register_udf._judf)：将 Python UDF 注册到 Spark SQL 中，使其可以在 SQL 查询中使用。

可以看到手动注册的函数比装饰器多一步，那就是往sql注册，这就是为什么dsl能用，但sql却用不了。

装饰器技巧-functools.partial

​ 看过上一篇装饰器语法糖规则的都应该知道，如果装饰器在第一个括号里面传入参数，就是说明是在对闭包函数内层传参，那么这里并没有看见内嵌函数，包括源码里面的装饰器都没有看见内嵌函数。

​ 这是因为在 pyspark 的 udf 实现中，它使用了一个巧妙的技巧：如果 f 是 None 或者 f 是一个字符串或 DataType 实例，它返回一个部分应用（partial application）的 _create_udf 函数。这种技巧允许 udf 函数既能直接装饰函数，也能作为一个带参数的装饰器来使用。

functools.partial 是 Python 的一个高阶函数，用于部分应用一个函数的参数。它返回一个新的函数，这个新函数预设了部分参数值，并等待其他参数传入。
